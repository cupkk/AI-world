## 3) API 联调用例

### 3.1 Hub / Talent

- [ ] `GET /api/hub` 返回 `items` 或数组时，Hub 正常展示
- [ ] `GET /api/hub` 不可用时，Hub 回退本地数据并提示降级
- [ ] `GET /api/talent` 返回 `items` 或数组时，Talent 正常展示
- [ ] `GET /api/talent` 不可用时，Talent 回退本地数据并提示降级

### 3.2 Auth / Invite

- [ ] `POST /api/auth/invite/verify` 成功后跳转注册态
- [ ] 邀请码校验接口失败时，前端回退本地校验逻辑
- [ ] `POST /api/auth/login` 成功后写入登录态并跳转 `/hub`
- [ ] 登录接口失败时，前端回退 demo 登录并提示降级

### 3.3 发布与审核

- [ ] `POST /api/publish` 创建草稿成功
- [ ] `POST /api/publish/:id/submit` 提交审核成功
- [ ] `GET /api/publish/mine` 能正确回显我的发布列表
- [ ] `GET /api/admin/review` 管理员待审列表正确
- [ ] `POST /api/admin/review/:id/approve` 审核通过后前端状态更新
- [ ] `POST /api/admin/review/:id/reject` 审核驳回后显示驳回原因
- [ ] 以上任一接口失败时，前端本地兜底流程可继续

### 3.4 消息 REST + WebSocket

- [ ] `GET /api/messages/conversations` 正常加载会话列表
- [ ] `GET /api/messages/requests` 正常加载消息请求
- [ ] `POST /api/messages/conversations` 创建/获取会话成功
- [ ] `GET /api/messages/conversations/:cid/messages` 拉取历史成功
- [ ] `POST /api/messages/conversations/:cid/messages` 发送消息成功
- [ ] `POST /api/messages/conversations/:cid/read` 已读同步成功
- [ ] `POST /api/messages/requests/:id/accept` 生效并可聊天
- [ ] `POST /api/messages/requests/:id/reject` 生效并移出会话
- [ ] `message:new` 事件可实时追加消息
- [ ] `conversation:update` 事件可实时更新未读数/状态

## 4) 页面状态机回归

### 4.1 发布状态机

- [ ] DRAFT：可编辑，不出现在 Hub
- [ ] PENDING_REVIEW：只读并显示审核中
- [ ] PUBLISHED：出现在 Hub / 详情可访问
- [ ] REJECTED：显示原因，可修改后重新提交

### 4.2 私信状态机

- [ ] 首次私信进入 PENDING 请求态

- [ ] 企业发布命中招聘关键词时前端阻断提交
