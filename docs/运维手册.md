# AI-World 运维手册

更新时间：2026-03-13

## 1. 部署架构

当前线上采用单台 ECS + Docker Compose 双栈部署：

- production: `server/docker-compose.prod.yml`
- staging: `server/docker-compose.staging.yml`
- edge: `server/ops/docker-compose.edge.yml`
- observability: `server/ops/docker-compose.observability.yml`

域名规划：

- production
  - `https://ai-world.asia`
  - `https://www.ai-world.asia`
- staging
  - `https://staging.ai-world.asia`

公网仅暴露：

- `GET /ready`
- `GET /health`

`/metrics` 不对公网开放，只允许 Prometheus 通过 Docker 内网抓取。

## 2. 服务器目录

服务器项目目录：

```bash
/opt/aiworld
```

常用工作目录：

```bash
cd /opt/aiworld/server
```

备份目录：

```bash
/opt/aiworld/backups/production
/opt/aiworld/backups/staging
```

## 3. 上线前前置项

必须确认以下条件已满足：

1. ECS 已绑定固定 EIP
2. DNS 已生效
   - `@ -> <EIP>`
   - `www -> <EIP>`
   - `staging -> <EIP>`
3. 证书已覆盖
   - `ai-world.asia`
   - `www.ai-world.asia`
   - `staging.ai-world.asia`
4. GitHub Actions environment secrets 已配置
5. 生产与 staging 的 SMTP / OSS / LLM 凭据已配置

Secrets 详情见 [GitHub_Actions_Secrets.md](/d:/github/AI-world/docs/GitHub_Actions_Secrets.md)。

## 4. 环境文件

首次部署前复制模板：

```bash
cd /opt/aiworld/server
cp .env.production.example .env.production
cp .env.staging.example .env.staging
```

production 与 staging 必须同时满足：

- `REQUIRE_OSS=true`
- `REQUIRE_SMTP=true`
- `REQUIRE_LLM=true`
- `ENABLE_DEMO_INVITES=false`
- `SEED_INCLUDE_DEMO_DATA=false`

前端构建必须关闭 demo：

- `VITE_ENABLE_DEMO_MODE=false`
- `VITE_ALLOW_DEMO_PREFILL=false`

## 5. 基础容器

首次部署执行：

```bash
cd /opt/aiworld/server
chmod +x ./scripts/*.sh
./scripts/ensure-docker-networks.sh
docker compose -f ./ops/docker-compose.edge.yml up -d
docker compose -f ./ops/docker-compose.observability.yml up -d
```

监控检查：

```bash

```

预期抓取目标：

- `aiworld-production-api`
- `aiworld-staging-api`

## 7. 标准发布流程

### 7.1 Deploy Staging

发布顺序固定为：

1. `CI`
2. `Deploy Staging`
3. staging 自动验收
4. staging 人工验收
5. `Promote Production`

不要跳过 staging，不要在服务器手工重建镜像。

### 7.2 Staging 自动验收

`Deploy Staging` 必须通过：

- staging 部署
- readiness / health
- observability
- `tests/live/prod-smoke.spec.ts`
- `tests/live/staging-mutation.spec.ts`

### 7.3 Staging 人工验收

按 [上线运营清单.md](/d:/github/AI-world/docs/%E4%B8%8A%E7%BA%BF%E8%BF%90%E8%90%A5%E6%B8%85%E5%8D%95.md) 执行。

### 7.4 Promote Production

`Promote Production` 的 `image_tag` 必须填写刚刚通过 staging 的 commit SHA。

不允许：

- 使用新的 tag
- 在生产机重新 `docker build`
- 跳过 production smoke

## 8. 手工运维命令

### 8.1 部署 staging

```bash
cd /opt/aiworld/server
REGISTRY=<registry> IMAGE_TAG=<tag> ./scripts/deploy-stack.sh staging
./scripts/check-stack.sh staging
```

### 8.2 部署 production

```bash
cd /opt/aiworld/server
REGISTRY=<registry> IMAGE_TAG=<tag> ./scripts/deploy-stack.sh production
./scripts/check-stack.sh production
```

### 8.3 关闭 staging

```bash
cd /opt/aiworld/server
docker compose -p aiworld-staging -f ./docker-compose.staging.yml down
```

### 8.4 回滚 production

```bash
cd /opt/aiworld/server
REGISTRY=<registry> ./scripts/rollback-stack.sh production <previous-tag>
./scripts/check-stack.sh production
```

回滚后必须重新执行：

- `https://ai-world.asia/ready`
- `https://ai-world.asia/health`
- production smoke


### 9.4 Staging mutation

```bash
set STAGING_BASE_URL=https://staging.ai-world.asia
set PLAYWRIGHT_BASE_URL=https://staging.ai-world.asia
set LIVE_ALLOW_MUTATIONS=1
set LIVE_ADMIN_EMAIL_STAGING=<admin-email>
set LIVE_ADMIN_PASSWORD_STAGING=<admin-password>
set LIVE_EXPERT_EMAIL_STAGING=<expert-email>
set LIVE_EXPERT_PASSWORD_STAGING=<expert-password>
set LIVE_ENTERPRISE_EMAIL_STAGING=<enterprise-email>
set LIVE_ENTERPRISE_PASSWORD_STAGING=<enterprise-password>
npm run test:live:staging-mutation
```

说明：

- staging mutation 中 learner 账号会自动注册新的测试用户，不再依赖固定 learner 账号状态。
- production 只跑 smoke，不跑 mutation。

## 11. 发布后 24 小时观测

发布后每 4 到 6 小时检查一次：

- `aiworld-edge`
- `aiworld-production-api`
- `aiworld-production-worker`
- Prometheus targets 是否 `up`
- AI Assistant 503 比例
- 注册、登录、发布、知识库上传的 4xx/5xx

首日如出现高频 5xx、消息异常、上传异常，先冻结发版，再进入修复。

## 13. 常见排障

### 13.1 API 不健康

```bash
cd /opt/aiworld/server
docker compose -p aiworld-production -f ./docker-compose.prod.yml ps
docker compose -p aiworld-production -f ./docker-compose.prod.yml logs api --tail 200
docker compose -p aiworld-staging -f ./docker-compose.staging.yml ps
docker compose -p aiworld-staging -f ./docker-compose.staging.yml logs api --tail 200
```

### 13.2 staging 域名或证书异常

优先检查：

1. `staging.ai-world.asia` 是否解析到固定 EIP
2. `edge` 容器是否运行
3. 证书是否已覆盖 `staging.ai-world.asia`
4. `./scripts/issue-certificate.sh --dry-run ...` 是否通过

### 13.3 `/metrics` 公网访问失败

这是预期行为。`/metrics` 不对公网开放。

请通过以下命令检查：

```bash
cd /opt/aiworld/server
./scripts/verify-observability.sh
```

### 13.4 AI Assistant 返回 503

检查：

1. `REQUIRE_LLM=true`
2. `LLM_API_KEY` 是否已配置
3. 上游模型服务是否可用

错误响应格式：

```json
{
  "code": 503,
  "errorCode": "ASSISTANT_UNAVAILABLE",
  "message": "..."
}
```
