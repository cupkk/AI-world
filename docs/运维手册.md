# AI-World 运维手册

更新时间：2026-03-17

## 1. 当前部署模型

当前采用单台 ECS 直接承载生产环境，不再维护独立 `staging` 栈。

- 域名：
  - `https://ai-world.asia`
  - `https://www.ai-world.asia`
- 服务器：
  - `root@47.238.143.212`
  - 项目目录：`/opt/aiworld`
- 运行中的 Compose 组件：
  - `server/docker-compose.prod.yml`
  - `server/ops/docker-compose.edge.yml`
  - `server/ops/docker-compose.observability.yml`

## 2. 目录约定

- 应用代码：`/opt/aiworld`
- 生产环境变量：`/opt/aiworld/server/.env.production`
- 备份目录：`/opt/aiworld/backups/production`
- TLS 证书：`/etc/letsencrypt`

## 3. 上线前硬要求

以下项目不满足时，不允许上线：

- `SESSION_SECRET` 为强随机值，不能使用默认值
- `ENABLE_DEMO_INVITES=false`
- `ENABLE_PUBLIC_SAMPLE_INVITES=false`
- `SEED_INCLUDE_DEMO_DATA=false`
- 前端构建关闭：
  - `VITE_ENABLE_DEMO_MODE=false`
  - `VITE_ALLOW_DEMO_PREFILL=false`
  - `VITE_ENABLE_PUBLIC_SAMPLE_INVITES=false`
- Assistant / Knowledge Base 必须显式配置
  - 前端：`VITE_ENABLE_ASSISTANT`、`VITE_ENABLE_KNOWLEDGE_BASE`
  - 后端：`ENABLE_ASSISTANT`、`ENABLE_KNOWLEDGE_BASE`
- 如果开启 Assistant：
  - `REQUIRE_LLM=true`

上线前先执行：

```bash
npm run release:preflight:production
```

如需指定真实环境文件：

```bash
node ./scripts/release-preflight.mjs --mode production --frontend-file .env.production --backend-file server/.env.production
```

## 4. 标准发布流程

### 4.1 本地校验

```bash
npm exec tsc --noEmit
npm run build
npm --prefix server test -- --runInBand
npm run release:preflight:production
```

### 4.2 同步代码到服务器

建议将当前工作区打包后同步到 `/opt/aiworld`，避免依赖未提交代码之外的状态。

### 4.3 服务器部署

在服务器 `/opt/aiworld/server` 目录执行：

```bash
chmod +x ./scripts/*.sh
./scripts/ensure-docker-networks.sh
docker compose -f ./ops/docker-compose.edge.yml up -d
docker compose -f ./ops/docker-compose.observability.yml up -d
REGISTRY=<registry> IMAGE_TAG=<tag> ./scripts/deploy-stack.sh
./scripts/check-stack.sh
./scripts/verify-observability.sh
```

如果镜像已经在服务器本地构建完成：

```bash
REGISTRY=<registry> IMAGE_TAG=<tag> SKIP_PULL=1 ./scripts/deploy-stack.sh
```

### 4.4 发布后验收

```bash
set PRODUCTION_BASE_URL=https://ai-world.asia
set PLAYWRIGHT_BASE_URL=https://ai-world.asia
set LIVE_ENABLE_ASSISTANT=0

set LIVE_ADMIN_PASSWORD=<admin-password>
set LIVE_LEARNER_EMAIL=<learner-email>
set LIVE_LEARNER_PASSWORD=<learner-password>

set LIVE_ENTERPRISE_EMAIL=<enterprise-email>
set LIVE_ENTERPRISE_PASSWORD=<enterprise-password>

```

## 5. 回滚

在服务器 `/opt/aiworld/server` 目录执行：

```bash
REGISTRY=<registry> ./scripts/rollback-stack.sh <previous-tag>
./scripts/check-stack.sh
```

只有在上一个镜像 tag 已知可用时才执行回滚。

## 8. 常见故障

### 8.1 `invite/samples` 仍然可访问

优先检查：

- 后端是否已部署到最新镜像
- `server/.env.production` 中是否显式设置 `ENABLE_PUBLIC_SAMPLE_INVITES=false`
- 前端是否已重新构建并发布

### 8.2 `/ready` 不通过

优先检查：

- `docker compose -p aiworld-production -f ./docker-compose.prod.yml ps`
- `docker compose -p aiworld-production -f ./docker-compose.prod.yml logs api --tail 200`
- PostgreSQL / Redis 健康状态
- Prisma migration 是否成功执行

### 8.3 站点打开异常

优先检查：

- `docker ps --format 'table {{.Names}}\t{{.Status}}\t{{.Ports}}'`
- `docker logs aiworld-edge --tail 200`
- `nginx-domain.conf` 是否只指向 `production-web` 和 `production-api`
- 证书是否覆盖 `ai-world.asia` 与 `www.ai-world.asia`

## 9. 安全提醒

当前仓库和服务器外仍需单独管理并定期轮换：

- SMTP 凭据
- OSS 凭据
- LLM API Key
- 服务器 SSH 私钥

这些凭据不应继续长期保留在服务器明文环境文件中而不轮换。
