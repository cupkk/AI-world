# 上线运营清单

## 1. 发布前

- 确认 `CI` 已通过。
- 确认本地提交中不包含 `playwright-report/`、`playwright-report-live/`、`test-results/`。
- 确认本次发布 commit 已合入 `main`。
- 记录待发布的 commit SHA，后续 `Deploy Staging` 和 `Promote Production` 必须使用同一 SHA。

## 2. Staging 自动门禁

在 GitHub Actions 中执行：

1. `Deploy Staging`
2. 确认以下检查全部为绿：
   - 镜像构建与推送
   - staging 部署
   - readiness / health
   - observability 检查
   - `tests/live/prod-smoke.spec.ts`
   - `tests/live/staging-mutation.spec.ts`

如果任一失败，停止发布，不进入 production。

## 3. Staging 人工验收

逐项验收并勾选：

- 邀请页能看到示例邀请码，注册流程可完成。
- learner 登录后可以发布内容。
- learner 可以上传知识库文件。
- AI Assistant 返回真实推荐结果。
- admin 在审核队列里能看到新提交内容。
- 消息页可进入且列表可见。
- 企业工作台可进入且页面无报错。

建议验收地址：

- `https://staging.ai-world.asia/login?tab=register`
- `https://staging.ai-world.asia/publish`
- `https://staging.ai-world.asia/settings/knowledge-base`
- `https://staging.ai-world.asia/assistant`
- `https://staging.ai-world.asia/admin/review`
- `https://staging.ai-world.asia/messages`
- `https://staging.ai-world.asia/app/enterprise`

## 4. Promote Production

在 GitHub Actions 中执行：

1. 运行 `Promote Production`
2. `image_tag` 只填写刚刚通过 staging 的 commit SHA
3. 不允许替换成其他 tag

## 5. Production 验收

`Promote Production` 完成后，立即检查：

- `https://ai-world.asia/ready`
- `https://ai-world.asia/health`
- `npm run test:live:prod-smoke`

人工抽查：

- 首页可访问
- 登录页可访问
- 邀请页可访问
- 发布页可访问
- 知识库页可访问

如果 production smoke 失败，立即执行镜像 tag 回滚，不做在线修补。

## 6. 24 小时观测

发布后每 4 到 6 小时人工检查一次：

- `aiworld-edge`、`aiworld-production-api`、`aiworld-production-worker` 容器状态
- Prometheus targets 是否仍为 `up`
- AI Assistant 503 比例
- 注册、登录、发布、知识库上传的 4xx/5xx

建议每次抽查的用户路径：

- 登录
- 注册
- 发布
- 审核
- 私信
- 知识库

如果首日出现高频 5xx、消息异常、上传异常，先冻结发版，再进入修复。

## 7. 本周必须收尾

- 轮换部署 SSH key
- 轮换 SMTP / OSS / LLM 等历史密钥
- 修正文档编码与终端可读性
- 记录每次上线的 commit SHA、image tag、发布时间、回滚 tag、验收结果
