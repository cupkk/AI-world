## 一、认证 & 准入

| # | 用例 | 验收标准 | 状态 |
|---|------|---------|------|
| 1.1 | 邀请码验证 | `POST /api/auth/invite/verify` 返回邀请码状态；无效码返回 4xx | 🔲 |
| 1.2 | 注册 | `POST /api/auth/register` 创建用户，返回 session；重复邮箱返回 409 | 🔲 |
| 1.3 | 登录 | `POST /api/auth/login` 写入 session；密码错误返回 401 | 🔲 |
| 1.4 | 登出 | `POST /api/auth/logout` 销毁 session；后续请求返回 401 | 🔲 |
| 1.5 | 忘记密码 | `POST /api/auth/forgot-password` 发送重置邮件（或返回 token） | 🔲 |
| 1.6 | 重置密码 | `POST /api/auth/reset-password` 使用 token 重置密码，旧密码失效 | 🔲 |
| 1.7 | 全站登录拦截 | 未登录访问 `/hub`、`/talent`、`/messages` 均跳转 `/login` | ✅ |
| 1.8 | Onboarding 拦截 | 新用户登录后若未完成 Onboarding，强制跳转 `/onboarding` | 🔲 |

## 二、发布 → 审核完整链路（P0 重点）

| # | 用例 | 验收标准 | 状态 |
|---|------|---------|------|
| 2.1 | 创建草稿 | `POST /api/publish` → status=DRAFT；description→summary 映射正确 | ✅ |
| 2.2 | 编辑草稿 | `PATCH /api/hub/:id` → 仅 DRAFT/REJECTED 可编辑；summary/type/tags 更新生效 | ✅ |
| 2.3 | 提交审核 | `POST /api/publish/:id/submit` → status 变为 PENDING_REVIEW | ✅ |
| 2.4 | 管理员审批 | `POST /api/admin/review/:id/approve` → status 变为 PUBLISHED，publishedAt 写入 | 🔲 |
| 2.5 | 管理员驳回 | `POST /api/admin/review/:id/reject` → status 变为 REJECTED，rejectReason 写入 | 🔲 |
| 2.6 | 被拒后重新编辑 | REJECTED 状态调用 PATCH → 可编辑 | ✅ |
| 2.7 | 被拒后重新提交 | REJECTED 状态调用 submit → status 变 PENDING_REVIEW，rejectReason 清空 | ✅ |
| 2.8 | 我的发布列表 | `GET /api/publish/mine` → 返回当前用户所有内容（含 DRAFT/PENDING/PUBLISHED/REJECTED） | ✅ |
| 2.9 | 类型校验 | 无效 type（如 "INVALID"）→ 返回 400（非 500） | ✅ |
| 2.10 | 招聘内容拦截 | 企业用户发布含招聘关键词内容 → 前端阻止提交 + 后端二次校验 | ✅ |
